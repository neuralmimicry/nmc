SSH Key API Endpoints

Manage SSH public keys for secure access

← Back to API Home

Create SSH Key

Endpoint: POST /ssh/create

Creates a new SSH public key entry, associating a name and public key string.

Request Body:


{
    "name": "my-ssh-key-prod",
    "publicKey": "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQ...",
    "description": "SSH key for production servers"
}

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "SSH key 'my-ssh-key-prod' created successfully.",
    "data": {
        "id": "sshkey-abc123def456",
        "name": "my-ssh-key-prod",
        "publicKey": "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAAQ...",
        "description": "SSH key for production servers"
    }
}

Error Response (HTTP 400 Bad Request if parameters missing):


{
    "success": false,
    "message": "Missing required parameters: name or publicKey.",
    "data": {}
}

Error Response (HTTP 409 Conflict if key with name already exists):


{
    "success": false,
    "message": "SSH key 'my-ssh-key-prod' already exists.",
    "data": {}
}

Delete SSH Key

Endpoint: DELETE /ssh/delete/{id}

Deletes a specific SSH key by its ID.

Path Parameters:

  • id (string): The unique ID of the SSH key to delete.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "SSH key 'sshkey-abc123def456' deleted successfully.",
    "data": null
}

Error Response (HTTP 404 Not Found):


{
    "success": false,
    "message": "SSH key 'non-existent-id' not found.",
    "data": {}
}

List SSH Keys

Endpoint: GET /ssh/list

Lists all registered SSH keys. Supports optional filtering by name.

Query Parameters:

  • filter-name (optional, string): Filters keys whose names contain the provided string.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "SSH keys listed successfully.",
    "data": [
        {
            "id": "sshkey-abc123def456",
            "name": "my-ssh-key-prod",
            "publicKey": "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAAQ...",
            "description": "SSH key for production servers"
        },
        {
            "id": "sshkey-xyz789uvw012",
            "name": "dev-ssh-key",
            "publicKey": "ssh-rsa BBBBB3NzaC1yc2EAAAADAQABAAAB...",
            "description": "Key for development instances"
        }
    ]
}

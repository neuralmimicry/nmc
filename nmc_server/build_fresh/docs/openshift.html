OpenShift API Endpoints

List OpenShift Resources

Endpoint: GET /openshift/resources

Returns a snapshot of available OpenShift capacity from the NeuralMimicry OpenShift portal API.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "OpenShift resources retrieved.",
    "data": {
        "total_gpus": 128,
        "available_gpus": 42,
        "nodes": {
            "intel": 20,
            "arm": 15
        },
        "storage_tb": 500
    }
}

List OpenShift Clusters

Endpoint: GET /openshift/clusters

Returns the list of clusters known to the OpenShift portal API.

Status values are normalized to common NMC states: Pending, Provisioning, Ready, Failed, Unknown.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "OpenShift clusters listed.",
    "data": [
        {
            "name": "hpc-cluster-prod",
            "status": "Ready",
            "endpoint": "https://api.hpc-cluster-prod.example.com:6443"
        }
    ]
}

Request OpenShift Cluster

Endpoint: POST /openshift/clusters/request

Submits a provisioning request to the OpenShift portal API.

Tracey policy: requests must include a tracey block so the cluster is tracked against its required Tracey agent.

Request Body:


{
    "name": "hpc-example",
    "organization": "neuralmimicry",
    "gpu_count": 8,
    "architecture": "amd64",
    "region": "us-east-1",
    "provider": "rosa",
    "burst_targets": ["aro", "gcp"],
    "tracey": {
        "agent_id": "tracey-hpc-example",
        "status_addr": "https://10.20.0.17:48000"
    }
}

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "OpenShift cluster request submitted.",
    "data": {
        "message": "ROSA Cluster hpc-example provisioning started via Terraform",
        "cluster": {
            "name": "hpc-example",
            "status": "Provisioning",
            "endpoint": null,
            "provider": "rosa"
        },
        "tracey": {
            "required": true,
            "agent_id": "tracey-hpc-example",
            "status_addr": "https://10.20.0.17:48000",
            "compliance_state": "pending"
        }
    }
}

Error Response (HTTP 400 Bad Request):


{
    "success": false,
    "message": "Missing required parameters: name, organization, gpu_count, architecture, or region.",
    "data": {}
}

Status Polling (CLI)

The CLI can poll cluster status using the proxy list endpoint:


./nmc openshift status hpc-example --watch --until Ready --interval 10 --timeout 900

Under the hood, this repeats GET /openshift/clusters until the target cluster reaches the requested --until state or the timeout expires.

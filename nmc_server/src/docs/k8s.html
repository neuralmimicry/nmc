Kubernetes (K8s) API Endpoints

Create K8s Cluster

Endpoint: POST /k8s/create

Provisions a new Kubernetes cluster.

Tracey policy: Continuum-managed clusters must include a tracey block identifying the required Tracey agent.

Request Body:


{
    "name": "my-prod-cluster",
    "region": "rugby-1",
    "tracey": {
        "agent_id": "tracey-my-prod-cluster",
        "status_addr": "http://10.42.0.55:48000"
    }
}

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "K8s cluster 'my-prod-cluster' created successfully.",
    "data": {
        "id": "k8s-abc123def456",
        "name": "my-prod-cluster",
        "region": "rugby-1",
        "status": "Provisioning"
    }
}

Error Response (HTTP 400 Bad Request if parameters missing):


{
    "success": false,
    "message": "Missing required parameters: name or region.",
    "data": {}
}

Error Response (HTTP 409 Conflict if cluster exists):


{
    "success": false,
    "message": "K8s cluster 'my-prod-cluster' already exists.",
    "data": {}
}

Delete K8s Cluster

Endpoint: DELETE /k8s/delete/{id}

Deletes a specific Kubernetes cluster by its ID.

Path Parameters:

  • id (string): The unique ID of the Kubernetes cluster to delete.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "K8s cluster 'k8s-abc123def456' deleted successfully.",
    "data": null
}

Error Response (HTTP 404 Not Found):


{
    "success": false,
    "message": "K8s cluster 'non-existent-id' not found.",
    "data": {}
}

Get KubeConfig

Endpoint: GET /k8s/get-config/{id}

Retrieves the KubeConfig YAML for a specific Kubernetes cluster, allowing CLI tools (like kubectl) to connect.

Path Parameters:

  • id (string): The unique ID of the Kubernetes cluster.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "KubeConfig retrieved successfully.",
    "data": "apiVersion: v1\nclusters:\n- cluster:\n    server: https://kube.k8s-abc123def456.aarnn.network\n    certificate-authority-data: REDACTED\n  name: my-prod-cluster\ncontexts:\n- context:\n    cluster: my-prod-cluster\n    user: my-prod-cluster-user\n  name: my-prod-cluster-context\ncurrent-context: my-prod-cluster-context\nkind: Config\npreferences: {}\nusers:\n- name: my-prod-cluster-user\n  user:\n    token: REDACTED-TOKEN-FOR-my-prod-cluster\n"
}

Error Response (HTTP 404 Not Found):


{
    "success": false,
    "message": "K8s cluster 'non-existent-id' not found.",
    "data": {}
}

List K8s Clusters

Endpoint: GET /k8s/list

Lists all available Kubernetes clusters. Supports optional filtering by name.

Query Parameters:

  • filter-name (optional, string): Filters clusters whose names contain the provided string.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "K8s clusters listed successfully.",
    "data": [
        {
            "id": "k8s-abc123def456",
            "name": "my-prod-cluster",
            "region": "rugby-1",
            "status": "Running"
        },
        {
            "id": "k8s-xyz789uvw012",
            "name": "dev-cluster",
            "region": "london-1",
            "status": "Suspended"
        }
    ]
}

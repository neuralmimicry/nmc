NMC Server API Documentation

Comprehensive guide to available API endpoints

API Categories

  • Bucket API - Manage cloud storage buckets.
  • Kubernetes (K8s) API - Control Kubernetes clusters.
  • Model API - Upload and manage machine learning models.
  • SSH Key API - Manage SSH public keys.
  • Virtual Machine (VM) API - Provision and manage virtual machines.

Introduction

Welcome to the NMC Server API Documentation. This server provides a simulated environment for various cloud resource management operations, including storage buckets, Kubernetes clusters, machine learning models, SSH keys, and virtual machines.

This documentation details each available API endpoint, including the HTTP method, request parameters, example request bodies, and expected successful and error responses. Use these examples to understand how to interact with the server.

Base URL

All API calls should be made relative to the server's base URL, which is typically http://localhost:8080 (or the host and port your server is running on).

General Response Structure

All successful API responses from this server will follow a consistent JSON structure:


{
    "success": true,
    "message": "A descriptive message about the operation.",
    "data": {
        // Relevant data for the specific operation (can be an object or array)
    }
}

Error responses will also follow a similar structure, but with "success": false and an appropriate HTTP status code:


{
    "success": false,
    "message": "A descriptive error message.",
    "data": {} // Typically an empty object for errors
}

HTTP Status Codes will indicate the general outcome of the request:

  • 200 OK: The request was successful.
  • 400 Bad Request: The request was malformed or missing required parameters.
  • 404 Not Found: The requested resource could not be found.
  • 409 Conflict: The request could not be completed due to a conflict (e.g., resource already exists).
  • 500 Internal Server Error: An unexpected error occurred on the server side.

VCluster API Endpoints

Manage virtual Kubernetes clusters within your host clusters

What is VCluster?

VClusters are fully functional virtual Kubernetes clusters that run inside namespaces of a host cluster. They provide strong isolation and allow teams to manage their own Kubernetes environments without requiring separate physical clusters.

Benefits:

  • Cost-effective: Multiple virtual clusters on a single host
  • Isolation: Each vcluster has its own API server and control plane
  • Fast provisioning: Deploys in seconds
  • Full Kubernetes API: Behaves like a real cluster

Create VCluster

Endpoint: POST /vcluster/create

Creates a new virtual Kubernetes cluster.

Request Body:


{
    "name": "my-vcluster",
    "namespace": "vcluster-my-vcluster"  // Optional, defaults to "vcluster-{name}"
}

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "VCluster 'my-vcluster' created successfully.",
    "data": {
        "id": "abc123def456",
        "name": "my-vcluster",
        "region": "vcluster",
        "status": "Provisioning",
        "is_vcluster": true,
        "parent_cluster": "https://127.0.0.1:6443",
        "vcluster_namespace": "vcluster-my-vcluster"
    }
}

Error Response (HTTP 400 Bad Request if name missing):


{
    "success": false,
    "message": "Missing required parameter: name.",
    "data": {}
}

Delete VCluster

Endpoint: DELETE /vcluster/delete/{id_or_name}

Deletes a specific virtual cluster by its ID or name.

Path Parameters:

  • id_or_name (string): The ID or name of the vcluster to delete.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "VCluster 'my-vcluster' deleted successfully.",
    "data": null
}

Error Response (HTTP 404 Not Found):


{
    "success": false,
    "message": "VCluster 'non-existent' not found.",
    "data": {}
}

List VClusters

Endpoint: GET /vcluster/list

Lists all available virtual clusters. Supports optional filtering by name.

Query Parameters:

  • filter-name (optional, string): Filters vclusters whose names contain the provided string.

    "message": "Successfully listed vclusters.",
    "data": {
        "vclusters": [

Get VCluster KubeConfig

Endpoint: GET /vcluster/get-config/{id_or_name}

Retrieves the KubeConfig for connecting to a specific vcluster.

Path Parameters:

  • id_or_name (string): The ID or name of the vcluster.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "VCluster kubeconfig retrieved successfully.",
    "data": {
        "kubeconfig": "apiVersion: v1\nclusters:\n- cluster:\n    server: https://my-vcluster.vcluster-my-vcluster.svc\n    certificate-authority-data: REDACTED\n  name: my-vcluster\ncontexts:\n- context:\n    cluster: my-vcluster\n    user: my-vcluster-user\n  name: my-vcluster-context\ncurrent-context: my-vcluster-context\nkind: Config\npreferences: {}\nusers:\n- name: my-vcluster-user\n  user:\n    token: REDACTED\n"
    }
}

Error Response (HTTP 404 Not Found):


{
    "success": false,
    "message": "VCluster kubeconfig secret not found.",
    "data": {}
}

VCluster Architecture

Each vcluster consists of:

  • StatefulSet: Runs the vcluster control plane (API server, controller manager, scheduler)
  • Service: Exposes the vcluster API server
  • PersistentVolume: Stores the vcluster's etcd data
  • Namespace: Isolates the vcluster resources in the host cluster

The vcluster synchronizes certain resources (pods, services, etc.) from the virtual cluster to the host cluster's namespace, providing isolation while leveraging the host cluster's compute resources.

Node Recruitment API Endpoint

Recruit remote Ubuntu hosts into Continuum using script/binary transfer with optional tenant auto-configuration

← Back to API Home

Recruit Node

Endpoint: POST /node/recruit

Auth: Requires the same Continuum API authentication as other protected endpoints (Bearer token or OIDC).

Transfers a script/binary over SSH (scp) and executes it on the remote host (ssh), then can run an Ansible playbook for tenant-specific host preparation.

Request Body:

{
  "host": "192.168.1.60",
  "user": "ubuntu",
  "port": 22,
  "node_type": "kubernetes",
  "auto_configure": true,
  "tenant_id": "acme",
  "tenant_name": "Acme Manufacturing",
  "tenant_environment": "prod",
  "capabilities": ["apps", "podman", "kubernetes"],
  "ansible_playbook": "/opt/nmc/ansible/recruited-node.yml",
  "ansible_extra_vars": {
    "nmc_ops_contact": "ops@acme.example",
    "nmc_backup_window": "02:00-03:00"
  },
  "script": "#!/usr/bin/env bash\nset -euo pipefail\necho recruited",
  "ssh_key_path": "/home/continuum/.ssh/id_ed25519",
  "remote_path": "/tmp/nmc-recruit.sh",
  "region": "home-lab",
  "dry_run": false,
  "sudo": true,
  "sudo_password": "example-password",
  "ansible_become": true,
  "tracey": {
    "agent_id": "tracey-192-168-1-60",
    "status_addr": "http://192.168.1.60:48000"
  }
}

Request Fields:

  • host (required): target hostname or IPv4 address.
  • user (optional): SSH username, default ubuntu.
  • port (optional): SSH port, default 22.
  • node_type (optional): bare-metal, app-install, vm, podman, kubernetes.
  • script or script_path (optional): recruitment script content/path on server.
  • binary_path (optional): binary path on server (mutually exclusive with script input).
  • binary_args (optional): string or string array of binary arguments.
  • ssh_key_path (optional): private key path on server.
  • remote_path (optional): destination path on remote node.
  • dry_run (optional): when true, returns planned commands without executing SSH/SCP.
  • sudo (optional): when true, executes artifact with sudo (uses sudo -S when password is supplied).
  • sudo_password / become_password (optional): password used for remote sudo and Ansible become.
  • auto_configure (optional): run post-recruitment Ansible configuration.
  • ansible_become / become (optional): enable/disable Ansible privilege escalation (default: true).
  • tenant_id, tenant_name, tenant_environment (optional): tenant metadata passed to Ansible.
  • capabilities / workloads (optional): requested host capabilities: apps, vm, podman, kubernetes.
  • ansible_playbook (optional): playbook path override. Falls back to NMC_RECRUIT_ANSIBLE_PLAYBOOK or ansible/recruited-node.yml.
  • ansible_extra_vars (optional): JSON object merged into Ansible -e vars.
  • recruit_token (optional): required if server sets NMC_RECRUIT_TOKEN.
  • tracey (optional/required by policy): Tracey managed-node requirement block.

Error Responses:

  • HTTP 400: invalid input (host/user/path/type/JSON fields).
  • HTTP 401: unauthorized request.
  • HTTP 403: invalid or missing recruit_token when required.
  • HTTP 500: required dependencies missing on recruiter host (for example ansible-playbook when auto_configure=true).
  • HTTP 502: SSH/SCP prepare/transfer/execute step failed.

Security note: Sensitive values such as continuum_auth_token and sudo_password are redacted from diagnostics and command output fields.

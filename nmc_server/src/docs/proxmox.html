Proxmox API Endpoints

List Proxmox Resources

Endpoint: GET /proxmox/resources

Returns a snapshot of available Proxmox capacity from the configured backend provider service.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "Proxmox resources retrieved.",
    "data": {
        "total_gpus": 24,
        "available_gpus": 10,
        "cluster_nodes": 4,
        "storage_tb": 96
    }
}

List Proxmox Clusters

Endpoint: GET /proxmox/clusters

Returns clusters known to the Proxmox orchestration backend.

Status values are normalised to NMC states: Pending, Provisioning, Ready, Failed, Unknown.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "Proxmox clusters listed.",
    "data": [
        {
            "name": "edge-pve-prod",
            "status": "Ready",
            "endpoint": "https://pve.edge.example.com:8006",
            "provider": "proxmox"
        }
    ]
}

Get Proxmox Cluster Details

Endpoint: GET /proxmox/details/{idOrName}

Returns details for one Proxmox cluster, including endpoint-derived hosts/IPs and ports in use.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "Proxmox cluster details retrieved.",
    "data": {
        "name": "edge-pve-prod",
        "status": "Ready",
        "endpoint": "https://pve.edge.example.com:8006",
        "networking": {
            "endpoint": "https://pve.edge.example.com:8006",
            "endpoint_candidates": [
                "https://pve.edge.example.com:8006"
            ],
            "ips_in_use": [
                "pve.edge.example.com"
            ],
            "ports_in_use": [
                8006
            ]
        }
    }
}

Request Proxmox Cluster

Endpoint: POST /proxmox/clusters/request

Submits a Proxmox cluster provisioning request to the configured backend provider API.

Provider options: proxmox, proxmox-ve, hybrid-burst.

Tracey policy: requests must include a tracey block when managed-resource compliance is required.

Request Body:


{
    "name": "edge-pve-example",
    "organization": "neuralmimicry",
    "gpu_count": 8,
    "architecture": "amd64",
    "region": "edge-1",
    "provider": "proxmox",
    "tracey": {
        "agent_id": "tracey-edge-pve-example",
        "status_addr": "https://10.20.0.37:48000"
    }
}

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "Proxmox cluster request submitted.",
    "data": {
        "cluster": {
            "name": "edge-pve-example",
            "status": "Provisioning",
            "provider": "proxmox"
        },
        "tracey": {
            "required": true,
            "agent_id": "tracey-edge-pve-example",
            "status_addr": "https://10.20.0.37:48000",
            "compliance_state": "pending"
        }
    }
}

Status Polling (CLI)

The CLI can poll cluster status via:


./nmc proxmox status edge-pve-example --watch --until Ready --interval 10 --timeout 900

Under the hood, this repeatedly calls GET /proxmox/details/{idOrName} until the target status is reached or timeout expires.

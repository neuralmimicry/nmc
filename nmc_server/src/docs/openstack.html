OpenStack API Endpoints

List OpenStack Resources

Endpoint: GET /openstack/resources

Returns a snapshot of available OpenStack capacity from the configured backend provider service.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "OpenStack resources retrieved.",
    "data": {
        "total_gpus": 64,
        "available_gpus": 21,
        "compute_nodes": 12,
        "storage_tb": 320
    }
}

List OpenStack Clusters

Endpoint: GET /openstack/clusters

Returns clusters known to the OpenStack orchestration backend.

Status values are normalized to NMC states: Pending, Provisioning, Ready, Failed, Unknown.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "OpenStack clusters listed.",
    "data": [
        {
            "name": "hpc-openstack-prod",
            "status": "Ready",
            "endpoint": "https://api.hpc-openstack-prod.example.com:6443",
            "provider": "openstack"
        }
    ]
}

Get OpenStack Cluster Details

Endpoint: GET /openstack/details/{idOrName}

Returns details for one OpenStack cluster, including endpoint-derived hosts/IPs and ports in use.

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "OpenStack cluster details retrieved.",
    "data": {
        "name": "hpc-openstack-prod",
        "status": "Ready",
        "endpoint": "https://api.hpc-openstack-prod.example.com:6443",
        "networking": {
            "endpoint": "https://api.hpc-openstack-prod.example.com:6443",
            "endpoint_candidates": [
                "https://api.hpc-openstack-prod.example.com:6443"
            ],
            "ips_in_use": [
                "api.hpc-openstack-prod.example.com"
            ],
            "ports_in_use": [
                6443
            ]
        }
    }
}

Request OpenStack Cluster

Endpoint: POST /openstack/clusters/request

Submits an OpenStack cluster provisioning request to the configured backend provider API.

Tracey policy: requests must include a tracey block when managed-resource compliance is required.

Request Body:


{
    "name": "hpc-openstack-example",
    "organization": "neuralmimicry",
    "gpu_count": 8,
    "architecture": "amd64",
    "region": "uk1",
    "provider": "openstack",
    "tracey": {
        "agent_id": "tracey-hpc-openstack-example",
        "status_addr": "https://10.20.0.27:48000"
    }
}

Successful Response (HTTP 200 OK):


{
    "success": true,
    "message": "OpenStack cluster request submitted.",
    "data": {
        "cluster": {
            "name": "hpc-openstack-example",
            "status": "Provisioning",
            "provider": "openstack"
        },
        "tracey": {
            "required": true,
            "agent_id": "tracey-hpc-openstack-example",
            "status_addr": "https://10.20.0.27:48000",
            "compliance_state": "pending"
        }
    }
}

Status Polling (CLI)

The CLI can poll cluster status via:


./nmc openstack status hpc-openstack-example --watch --until Ready --interval 10 --timeout 900

Under the hood, this repeatedly calls GET /openstack/details/{idOrName} until the target status is reached or timeout expires.

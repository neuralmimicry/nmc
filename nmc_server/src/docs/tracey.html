Tracey Agent API Documentation

Fleet, rack, server, and GPU telemetry for the Continuum self-healing control plane.

POST /tracey/agents/heartbeat

Registers or updates a Tracey agent heartbeat so the Continuum dashboard can track live health.

Example Request

curl -X POST http://localhost:8080/tracey/agents/heartbeat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "agent_id": "tracey-edge-01",
    "cluster": "k8s-prod-eu",
    "status": "healthy",
    "version": "1.5.2",
    "host": "node-17",
    "metrics": {
      "cpu_pct": 18.3,
      "mem_pct": 54.9,
      "queue_depth": 2
    }
  }'

Success Response

{
  "success": true,
  "message": "Tracey heartbeat accepted.",
  "data": {
    "agent_id": "tracey-edge-01",
    "cluster": "k8s-prod-eu",
    "status": "healthy",
    "version": "1.5.2",
    "host": "node-17",
    "metrics": {
      "cpu_pct": 18.3,
      "mem_pct": 54.9,
      "queue_depth": 2
    },
    "last_seen_epoch_ms": 1742389200000,
    "last_seen_seconds_ago": 0,
    "stale": false
  }
}

GET /tracey/agents

Returns all detected Tracey agents plus summary rollups used by the dashboard, including managed-resource compliance for Continuum-initiated workloads.

Success Response

{
  "success": true,
  "message": "Tracey agents listed successfully.",
  "data": {
    "summary": {
      "total": 3,
      "healthy": 2,
      "degraded": 1,
      "offline": 0,
      "stale": 0,
      "unknown": 0
    },
    "stale_after_seconds": 90,
    "agents": [
      {
        "agent_id": "tracey-edge-01",
        "cluster": "k8s-prod-eu",
        "status": "healthy",
        "version": "1.5.2",
        "host": "node-17",
        "metrics": {},
        "last_seen_epoch_ms": 1742389200000,
        "last_seen_seconds_ago": 3,
        "stale": false
      }
    ]
    }
}

Managed Resource Compliance

The response also includes:

  • tracey_policy - enforcement and grace settings.
  • requirement_summary - totals for compliant, pending, and noncompliant managed resources.
  • requirements - per-resource Tracey requirement state keyed by resource type/name.

Continuum bootstraps a default local requirement for its own server process (resource type continuum_server) so the first Tracey sidecar is expected to run alongside nmc_server.

Bootstrap controls: NMC_TRACEY_BOOTSTRAP_LOCAL_AGENT, NMC_TRACEY_LOCAL_AGENT_ID, NMC_TRACEY_LOCAL_STATUS_ADDR.

GET /tracey/fleet

Builds the fleet-wide Continuum operator view from cached Tracey status snapshots.

  • summary - agent, rack, zone, GPU, ECC, thermal, autonomy, and TraceyGuard rollups.
  • zone_breakdown - per-zone health and alert summaries.
  • racks - rack summaries used by the rack explorer.
  • gpu_heatmap - rack-grouped GPU tiles for fleet heatmap previews.
  • recent_actions - bounded autonomous action log across the fleet.

GET /tracey/adaptive

Builds the integrated plan / ramp / optimise / repeat loop used by the Continuum dashboard and CLI.

  • policy query parameter - optional placement bias: balanced, throughput, risk, or energy.
  • summary - fleet-wide mode, resolved placement policy, overall/readiness/placement scores, ramp pressure, placement counts, and next action.
  • plan, ramp, optimise, repeat - normalized phase snapshots for the always-on control loop.
  • recommendations - prioritized operator actions merged from per-agent loop state.
  • placement_candidates, gpu_candidates - best-fit host/GPU targets for upcoming workload placement.
  • recent_actions - fleet feedback signals that close the loop into the next scheduling cycle.

GET /tracey/cve/status

Reports Continuum's server-side CVE mirror and index state used for distributed Tracey compromise assessment.

  • repo_url, head, indexed_head
  • indexed_records - simplified searchable CVE corpus size
  • assessment_protocol_version - active Continuum/Tracey assessment wire contract
  • assessment_cycle_ms, default_slice_count, max_matches_per_report
  • last_success_ms, last_success_age_ms, last_error

GET /tracey/assessment/fleet

Returns fleet-wide compromise assessment progress and risk ordering across all Tracey agents.

  • mirror - CVE mirror state used by the assessment cycle
  • progress - expected/completed slices and cycle completion percentage
  • communication - fleet-wide rejected, duplicate, stale-plan, and semantic-fault totals
  • agents - risk-ranked agent rows with compromise score, CVE counts, KEV counts, and per-agent slice progress

GET /tracey/agents/{agent_id}/assessment/plan

Builds the server-generated assessment assignment for one Tracey agent.

  • agent_id path parameter - required Tracey agent identifier.
  • Returns the active mirror, protocol, slot, and work-slice information that the selected agent should process next.
  • Uses the current ordered fleet view so the same cycle can be partitioned consistently across agents.

POST /tracey/agents/{agent_id}/assessment/report

Accepts one agent's assessment slice report and folds it into the fleet-wide compromise cycle.

  • agent_id path parameter - required Tracey agent identifier.
  • Request body - JSON object generated by the agent from its assigned slice.
  • Invalid or non-object payloads are rejected with 400.

GET /tracey/racks

Returns rack summaries and zone breakdowns without the full per-agent analytics payload.

GET /tracey/racks/{rack}

Loads rack-level drilldown data including:

  • rack summary rollups
  • GPU slot heatmap cells
  • server list for the rack
  • recent actions scoped to the rack

GET /tracey/agents/{agent_id}/server

Returns full server telemetry for one Tracey agent.

  • server.thermal_sensors, fan_sensors, power_sensors
  • server.disks and server.processes
  • gpus enriched with TraceyGuard health overlays
  • recent_actions, recent_executions, local and remote faults

GET /tracey/agents/{agent_id}/gpus/{gpu_id}/telemetry

Returns deep GPU telemetry for one GPU on one Tracey agent.

  • summary - clocks, power, thermals, memory, ECC, guard state, reliability
  • sm_heatmap - per-SM execution/fault aggregation built from TraceyGuard probe history
  • probe_mix - execution volume by probe type
  • recent_executions - bounded probe execution history with runtime context
  • recent_faults, remote_faults, recent_actions

GET /tracey/agents/{agent_id}/compromise

Returns the per-agent compromise assessment view, including server-generated slice plan and the latest Tracey-local fuzzy risk summary.

  • plan - current slot timing, stable coordination token, late-joiner assignment mode, slice cadence, and completion state
  • progress - last submitted slice, inventory digest, request/disposition audit fields, and CVE match rollups
  • continuum_assessment - Tracey-local compromise snapshot with signals, recent matches, and recommended action
  • tracey_guard_summary, loader_threats - supporting local evidence feeds

POST /tracey/agents/{agent_id}/control

Forwards Tracey control requests to the selected Tracey agent.

Example Request

curl -X POST http://localhost:8080/tracey/agents/tracey-edge-01/control \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "enabled": true,
    "deep_dive": true,
    "tmr_enabled": true,
    "overhead_budget_pct": 2.5,
    "force_scan": true
  }'

GET /tracey/agents/{agent_id}/deepdive

Fetches the Tracey deep-dive snapshot (probe timeline, quarantines, local/remote fault signatures) for the selected agent.

nmc tracey fleet
nmc tracey adaptive
nmc tracey adaptive --policy risk
nmc tracey cve

nmc tracey assessment-report AGENT_ID --payload '{"slice_id":0,"result":"ok"}'
nmc tracey racks
nmc tracey rack RACK_ID
nmc tracey compromise AGENT_ID
nmc tracey server AGENT_ID
nmc tracey gpu AGENT_ID GPU_ID
